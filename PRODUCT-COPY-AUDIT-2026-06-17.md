# 产品稿与 demo 一致性审计 · 2026-06-17

本审计只记录当前发现的问题和修稿建议，未修改 demo 或产品稿正文。当前事实源以 `demo/feixiangai/校园版/` 可运行页面为准；`product-intro.html` 是产品背景介绍页，不纳入过稿体系。

## 结论

当前不一致主要来自三类：

1. 新增承接页没有进入过稿体系。
2. 页面本地 `review-template` 与 `assets/review-content.js` 兜底稿不同步。
3. 少量页面仍保留旧归属表述，需要和最新“校园版 / 个人版 / 飞象老师精选 / 本校内容”模型统一。

## 建议修稿顺序

1. 先补 `homework-insight.html`、`essay-insight.html` 的过稿入口和集中稿。
2. 再同步 `app-square.html`、`my-apps.html` 的双源漂移，优先统一收藏/归属规则。
3. 最后对 `index.html`、`chat.html` 做逐字段 diff，同步两份稿，避免破坏已经确认的输入框和知识库范围规则。
4. 修完后跑一次结构检查：所有非 `product-intro.html` 页面都应有 `review-template` 和 `REVIEW_CONTENT` 项，且两者 JSON 一致。

## 后续防漂移机制

每次 demo 迭代结束，增加一个固定收尾检查：

- 新增页面是否被首页、工作动态、详情页或导航链接到。
- 新增可达页面是否需要过稿；如果需要，是否同时写入本地模板和 `assets/review-content.js`。
- 已确认硬规则是否被写反：学校知识库命名、AI 录题来源、我的作品搜索/浏览数、收藏与双端同步、校园版 user 信息。
- 如果只改页面 UI，也要查是否有对应产品稿描述需要删除、降级或重写。
